```shell
# 创建 systemd socket 文件
sudo tee /etc/systemd/system/cri-dockerd.socket <<EOF
[Unit]
Description=CRI Dockerd Socket

[Socket]
ListenStream=/run/cri-dockerd.sock
SocketMode=0660
[Install]
WantedBy=sockets.target
EOF

# 创建 systemd service 文件
sudo tee /etc/systemd/system/cri-dockerd.service <<EOF
[Unit]
Description=CRI Dockerd Service
After=network.target docker.service
Requires=docker.service

[Service]
ExecStart=/usr/local/bin/cri-dockerd
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

# 这里一定一定给上面文件加上最大权限
cd /etc/systemd/system/
chmod 644 cri*

# 重新加载 systemd 配置
sudo systemctl daemon-reload

# 启动并设置开机自启
sudo systemctl enable --now cri-dockerd.socket
sudo systemctl enable --now cri-dockerd.service

# 检查状态和 socket 文件

systemctl status cri-dockerd.service
systemctl status docker
ls -l /run/cri-dockerd.sock
```

[注意]

这里100%会遇到这个`[kubelet-check] Initial timeout of 40s passed.`问题,这是因为`pause3.9`引起的，而我们执行初始化是需要`pause3.10`

[官方解释]

注意：
kubeadm init 出现 [kubelet-check] Initial timeout of 40s passed 不一定是失败。
如果后续超时失败，要先查 kubelet 日志。

本环境使用 kubeadm v1.28.2，kubeadm 推荐 pause:3.9。
如果 cri-dockerd 默认使用 registry.k8s.io/pause:3.10，而服务器无法访问 registry.k8s.io，
会导致 kubelet 创建 Pod sandbox 失败，进而 apiserver/etcd 无法启动。

正确做法：
固定 cri-dockerd 的 Pod sandbox 镜像为：
registry.aliyuncs.com/google_containers/pause:3.9

[解决方案]

```shell
# 编辑 cri-dockerd service：
sudo vi /etc/systemd/system/cri-dockerd.service

找到这一行：
ExecStart=/usr/local/bin/cri-dockerd
改成：
ExecStart=/usr/local/bin/cri-dockerd --pod-infra-container-image=registry.aliyuncs.com/google_containers/pause:3.9
```

步骤二:我选择的安装方式是`Docker + cri-dockerd`，使用 `cri-dockerd` 作为容器运行时接口。所以在`worker节点`加入集群时需要在最后添加上`--cri-socket=unix:///var/run/cri-dockerd.sock`

组装原理是: master节点输入`kubeadm token create --print-join-command`后输出的结果+加上+`--cri-socket=unix:///var/run/cri-dockerd.sock`

```shell
# 这就是加入进群的命令kubeadm join(需要在所有worker节点上执行)
kubeadm join 192.168.88.70:6443 --token rcxbri.s1oeuc8uyned9bxy \
  --discovery-token-ca-cert-hash sha256:b7f063897f14bcb5abc7d364fcde4b64633a72944a2b42729c1c76bcc797bc61 \
  --cri-socket=unix:///var/run/cri-dockerd.sock
```
